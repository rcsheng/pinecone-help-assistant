# Pinecone Help Assistant

A local demo app that showcases two approaches to building a RAG-powered Q&A system over Pinecone documentation.

## What it does

The app serves a single-page UI at `http://localhost:3000` with two modes:

### ⛓ RAG Pipeline
A hand-rolled retrieval pipeline:
1. Embeds the user query with **OpenAI** `text-embedding-3-small`
2. Runs an ANN search against a **Pinecone serverless index** (`pinecone-docs-demo`)
3. Feeds the top-k chunks as context to **Claude** (`claude-sonnet-4`) for answer synthesis with inline citations

### 🤖 Pinecone Assistant
A fully managed RAG service powered by the **Pinecone Assistant API**:
- Uploads scraped docs as plain-text files to a named assistant (`pinecone-help`)
- Supports **multimodal** ingestion — images, charts, and diagrams from PDFs are extracted, captioned, and made queryable
- Returns cited answers with no embedding pipeline or LLM key required

## Project structure

```
├── server.js                      # Express server — reads .env, proxies all API calls
├── pinecone_rag_demo_live.html    # Frontend UI (served at /)
│
├── ingest_pinecone_docs.py        # Scrapes Pinecone docs → chunks → upserts to Pinecone index (RAG mode)
├── pinecone_assistant_ingest.py   # Scrapes Pinecone docs → saves as .txt → uploads to Pinecone Assistant
├── pinecone_multimodal_ingest.py  # Uploads a PDF with multimodal=True for image/chart extraction
├── pinecone_assistant_chat.py     # Terminal REPL for chatting with the assistant
│
├── assistant_docs/                # Saved .txt files (generated by assistant ingest, gitignored)
└── .env                           # API keys (gitignored)
```

## Setup

### 1. Environment variables

Create a `.env` file in the project root:

```env
PINECONE_API_KEY=pcsk_...
OPENAI_API_KEY=sk-...          # Required for RAG mode only
ANTHROPIC_API_KEY=sk-ant-...   # Required for RAG mode only
```

Get your keys:
- Pinecone → [app.pinecone.io](https://app.pinecone.io)
- OpenAI → [platform.openai.com](https://platform.openai.com)
- Anthropic → [console.anthropic.com](https://console.anthropic.com)

### 2. Install dependencies

```bash
# Node server
npm install

# Python scripts
pip install "pinecone[assistant]>=8.0.0" openai requests beautifulsoup4 tiktoken python-dotenv
```

## Ingesting data

### RAG mode — scrape & embed into Pinecone index

```bash
python ingest_pinecone_docs.py
```

Scrapes 17 Pinecone documentation pages, chunks them (~400 words with overlap), embeds with OpenAI, and upserts into a serverless index named `pinecone-docs-demo`.

### Assistant mode — upload docs as files

```bash
python pinecone_assistant_ingest.py
```

Scrapes the same pages, saves each as a `.txt` file with metadata (`url`, `guides`, `section`, `slug`), and uploads to a Pinecone Assistant named `pinecone-help`. The assistant handles chunking and retrieval internally.

### Multimodal — upload a PDF with image extraction

```bash
python pinecone_multimodal_ingest.py
```

Uploads a PDF to the assistant with `multimodal=True`, enabling extraction of images, charts, and diagrams. Also runs a sample context query to show the returned text and image snippets.

## Running the app

```bash
npm start
# or for auto-restart on file changes:
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

The server resolves both the Pinecone index host and the Assistant host on startup and logs them to the console. The connection status bar in the UI shows which services are available.

---

## Terminal chat (Assistant mode)

```bash
python pinecone_assistant_chat.py
```

Interactive REPL with conversation history. Supports multimodal context (image captions from PDFs). Type `clear` to reset history, `quit` to exit.
