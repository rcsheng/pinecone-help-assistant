Evaluation Approach
Two evaluation layers run together on each question:
LLM-as-judge (Claude Sonnet) scores 4 quality dimensions (0–10). Reference answers provided for Answer Correctness.
Deterministic retrieval metrics (Hit Rate@1/3/5, MRR) computed by matching retrieved chunk sources against ground-truth document URLs confirmed from the live index.
CR Context Relevance weight 20% reference-free

Are the retrieved chunks relevant to the question? Judges retrieval quality without needing ground-truth document IDs.

1–3 — Chunks are off-topic or only about unrelated Pinecone features
4–6 — Chunks are about Pinecone but tangentially related to the question
7–10 — Chunks directly and completely address the question
Ground-truth source URLs are defined for all 10 questions. Hit Rate@1/3/5 and MRR are computed deterministically from chunk metadata — no LLM call needed.
F Faithfulness weight 35% reference-free

Does the answer stay grounded in the retrieved context without hallucinating? Each unsupported claim reduces the score.

1–3 — Answer introduces multiple facts absent from the retrieved chunks
4–6 — Answer mostly uses context but adds some unsupported claims
7–10 — Every claim in the answer is directly supported by a retrieved chunk
AC Answer Correctness weight 30% reference-based ✓

How correct and complete is the answer compared to the reference answer? The judge compares key facts, not exact wording.

1–3 — Answer misses most key facts from the reference or contradicts them
4–6 — Answer captures some key facts but omits important details
7–10 — Answer captures all key facts from the reference answer
AR Answer Relevance weight 15% reference-free

Does the generated answer actually address the question asked? Catches on-topic answers that still miss the point.

1–3 — Answer addresses a different question or refuses to answer
4–6 — Answer partially addresses the question but with significant drift
7–10 — Answer directly and fully addresses what was asked
Hit@K Hit Rate @ 1 / 3 / 5 deterministic ground-truth docs ✓

Did the correct source document appear in the top K retrieved chunks? Computed by matching chunk.metadata.source against the expected URL(s) defined for each question.

Miss — No retrieved chunk came from the expected document
Hit — At least one chunk from the expected document was retrieved
Expected sources confirmed by querying the live index for all 10 question topics.
MRR Mean Reciprocal Rank deterministic ground-truth docs ✓

Measures how high the first relevant chunk ranks. MRR = 1 / rank_of_first_hit (e.g., hit at rank 1 = 1.0, rank 2 = 0.5, rank 3 = 0.33). MRR = 0 if no relevant chunk is retrieved.

MRR 0.0 — Correct document not retrieved at all
MRR 0.33–0.5 — Correct document retrieved at rank 3 or 2
MRR 1.0 — Correct document is the top result
Not normalized (no NDCG): this index has too few total documents for NDCG to be meaningful.
LLM composite = CR × 0.20 + F × 0.35 + AC × 0.30 + AR × 0.15   |   Retrieval: Hit@1 / Hit@3 / Hit@5 / MRR (separate)
